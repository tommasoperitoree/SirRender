### `PFM` Format

example of a `PFM` file we had from the reference file used for testing (encoded _little endian_), `reference_le.pfm`:

0x50, 0x46, 0x0a, 0x33, 0x20, 0x32, 0x0a, 0x2d, 0x31, 0x2e, 0x30, 0x0a,
0x00, 0x00, 0xc8, 0x42, 0x00, 0x00, 0x48, 0x43, 0x00, 0x00, 0x96, 0x43,
0x00, 0x00, 0xc8, 0x43, 0x00, 0x00, 0xfa, 0x43, 0x00, 0x00, 0x16, 0x44,
0x00, 0x00, 0x2f, 0x44, 0x00, 0x00, 0x48, 0x44, 0x00, 0x00, 0x61, 0x44,
0x00, 0x00, 0x20, 0x41, 0x00, 0x00, 0xa0, 0x41, 0x00, 0x00, 0xf0, 0x41,
0x00, 0x00, 0x20, 0x42, 0x00, 0x00, 0x48, 0x42, 0x00, 0x00, 0x70, 0x42,
0x00, 0x00, 0x8c, 0x42, 0x00, 0x00, 0xa0, 0x42, 0x00, 0x00, 0xb4, 0x42

- as seen above, the content of a `PFM` file is in bytes, so we extract byte by byte until we get `0x0a` which is `\n`,
  or backspace
- the first two bytes, `0x50, 0x46` spell `PF` which is the tag that ensures we have a `.pfm` file
- the next line, up to the next backspace, is `0x33, 0x20, 0x32` which converts to `3 2` with the space, giving us our
  `height` and `width`
- lastly, the next 4 bytes, `0x2d, 0x31, 0x2e, 0x30` give the `-1.0`, i.e. the _little endian_ flag
- here the 4 Byte to 32-bit floating-point content begins, now we can get the content of our $3\times2=6$ pixels. Since
  we're using `r g b` colors, we need $3$ numbers each, thus $18$ total Floats, e.g. the first 4 Bytes are
  `0x00, 0x00, 0xc8, 0x42` $ = 100.0$, the `red` of the first pixel.
